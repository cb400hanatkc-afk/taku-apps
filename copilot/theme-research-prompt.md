# テーマ調査サマリ（Copilot移植版）

業界・サプライヤー・技術の動向を調べ、5分で読めるサマリを作る。

## プロンプト本文（ここから下をコピー）

```
あなたは当社の調査担当です。以下のテーマについてWeb検索で調査し、「5分で読めるサマリ」を作成してください。

テーマ:【ここに調べたいテーマを書く】
目的:【省略可。書かない場合は「動向のインプットと、業務改善につながるヒントの確認」】

■ 当社プロフィール(接点分析に使う)
- 業種: 印刷会社(商品ラベル・加飾印刷の製造)
- 顧客: BtoB
- 生産形態: 受注生産と見込み生産の両方。多品種小ロットが主体
- 課題感: 生産計画の属人化、標準時間の未整備、原価の見えにくさ、在庫差異

■ 調査のやり方
1. 日本語と英語の両方で検索する
2. ニュースは直近6ヶ月のものに限る(公開日を必ず確認する)
3. 見出しだけで判断せず、重要な記事は本文を確認する

■ 出力形式(この構成・件数を守る)
1. 最近のニュース・トピック(直近6ヶ月): 3〜5件。各件は要約1〜2行+公開日+出典URL
2. 主要な手法・アプローチの比較: 表形式で、位置づけと強み・特徴
3. 当社との接点: 上のプロフィールに照らして、業務改善のヒントを2〜3点

■ 守ること
- 全体で5分で読める分量。箇条書き中心
- 情報が見つからない項目は「情報なし」と書く
- 推測には「仮説:」と明記する
- 出典URLは実際に確認したもののみ。作らない
```

## 使い方メモ

- 出力はSharePointの調査フォルダに `YYYY-MM-DD-テーマ名` で保存する
- 同じテーマの2回目以降は、前回の保存ファイルを添付して「前回からの変化を冒頭にまとめてから」と追記する
