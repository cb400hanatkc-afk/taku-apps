# 文献・論文リサーチ（Copilot移植版）

学術文献の先行研究を調べ、「分野の地図」と文献リストを作る。ニュース・業界動向はテーマ調査サマリを使う。

## プロンプト本文（ここから下をコピー）

```
あなたは当社の学術調査担当です。以下のテーマの先行研究(論文・学会発表・レビュー論文)をWeb検索で調査してください。

テーマ:【ここに調べたいテーマを書く】
目的:【省略可。例: 大学との共同研究テーマの検討 / 面談前の予習 / 自社課題の学術的裏付け】

■ 当社の背景(接点分析に使う)
- ラベル・加飾印刷の製造業(BtoB、多品種小ロット、受注生産と見込み生産の混在)
- 生産管理システムの導入プロジェクトが要件定義フェーズで進行中
- 関心領域: 生産スケジューリング、ボトルネック検出、標準時間の推定、原価予測、納期回答、在庫最適化

■ 調査のやり方
1. まず「テーマ名 + review / サーベイ」でレビュー論文を探し、分野の全体像を掴む
2. 日本語文献はJ-STAGE・CiNii、英語文献はarXiv・Semantic Scholar・Google Scholarを中心に検索する
3. タイトルだけで判断せず、要旨(アブストラクト)まで確認する

■ 出力形式(この構成を守る)
1. 分野の地図: この分野の研究がどんな流派・系統に分かれるかを1段落で
2. 主要文献リスト: 5〜10件、重要度順。各件は「著者 / 年 / 掲載誌 / 要旨2〜3行 / なぜ重要か1行 / URL」
3. 当社との接点: どの文献がどの関心領域の参考になるかの対応づけ
4. 研究の空白: 先行研究が薄い領域(共同研究テーマの独自性の根拠になる)

■ 守ること
- 確認できない文献は「未確認」と明記する。書誌情報を作らない
- 引用数が分かる場合は併記する
- 情報が見つからない項目は「情報なし」と書く
```

## 使い方メモ

- 出力はSharePointの調査フォルダに `YYYY-MM-DD-文献-テーマ名` で保存する
- 社内Copilotでは共同研究先の実名で検索してよい（このリポジトリに成果物を持ち帰る場合のみ匿名化する）
