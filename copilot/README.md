# 社内Copilot 移植版プロンプト集

Claude Code のスキルを、社内の Microsoft 365 Copilot で使えるように移植したプロンプト集。

## 前提（2026-07時点の社内環境）

- Copilot Studio（エージェント作成）: **未許可** → 自動発動はできない。プロンプトはコピペで使う
- Web検索: **有効** → 調査系プロンプトはそのまま機能する
- 社内文書の参照: Copilot は SharePoint / OneDrive / Teams の文書を読める。ここが Claude Code 版より有利な点

## 収録プロンプト

| ファイル | 元スキル | 用途 |
|---|---|---|
| `theme-research-prompt.md` | theme-research | 業界・サプライヤー動向の調査サマリ |
| `paper-research-prompt.md` | paper-research | 学術文献・先行研究のリサーチ |
| `production-mgmt-prompt.md` | production-mgmt-implementation | 生産管理システム導入の相談相手 |
| `meeting-prep-prompt.md` | meeting-prep | 共同研究先との面談準備パック |
| （`../board/virtual-board-prompt.md`） | virtual-board | バーチャル役員会。既存の汎用移植版をそのまま使う（二重管理しない） |

## 使い方

1. 使いたいプロンプトのファイルを開き、コードブロック内を全部コピーする
2. 冒頭の【 】に今回の内容を書き込み、Copilot チャットに貼って送信する
3. よく使うものは Copilot の**プロンプトの保存**機能（Copilot Lab）に登録すると、コピペの手間が減る
4. 出力が途中で切れたら「続けて」と送る

## 運用ルール

- **成果物の保存先**: Copilot はファイルを保存できないので、出力を自分で SharePoint の所定フォルダに保存する。
  推奨: `調査サマリ/YYYY-MM-DD-テーマ名.docx` のように日付を頭に付けて統一する
- **継続性の作り方**: Copilot は過去のチャットや保存済みファイルを自動参照しない。
  前回の続きをやるときは、前回の保存ファイルを添付するか「/」でファイル参照してから依頼する
- **社内データの活用**: 社内Copilotでは匿名化不要。受注実績・会議録・実名入り資料をそのまま素材にできる。
  ただし**成果物をこの公開リポジトリに持ち帰る場合は、CLAUDE.md の秘匿ルールに従って匿名化してから**にすること

## 性能差への対策（プロンプト設計方針）

移植版は Claude 版より弱いモデルで動く前提で、次の方針で書いてある:

- 1プロンプト = 1タスク。長い多段作業は分割する
- 出力形式は見出し・件数まで明示する
- 「わからない場合は『情報なし』と書く」「出典URLは実在するもののみ」等の逃げ道と禁止事項を明記する
- 元スキルを更新したら、対応する移植版もあわせて更新すること（このREADMEの表が対応表）
