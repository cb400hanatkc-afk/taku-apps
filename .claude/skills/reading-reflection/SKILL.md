# 読書の振り返りスキル

Kindle注釈画面（ハイライト＋メモ）のスクリーンショットから、読書振り返りノートを生成する。

## 進め方

### フェーズ1: インプット受付
1. ユーザーは画像を複数回に分けて送ってくる。**「完了」と言われるまで分析・出力はしない。**
2. 各バッチの画像をReadで読み込み、以下を内部で記録する:
   - ハイライト本文とページ番号
   - **メモ（ユーザー自身の言葉）** — 最重要データ。ハイライトと区別して保持する
   - セクション見出し
3. 各バッチへの返信は1〜2行のみ:「◯枚目まで読み込みました（◯〜◯ページ分: 主なトピック）。続きをどうぞ。」
4. 重複画像があれば黙って除外し、完了時に軽く報告する。

### フェーズ2: 振り返り生成（「完了」の合図後）
`reading/YYYY-MM-DD-<書名スラッグ>.md` に以下の構成で保存する:

1. **ヘッダー**: 読了日・インプット枚数・ページ範囲・書名（不明なら「未確認」と明記し、ユーザーに確認）
2. **全体サマリ**: 本の狙いと、ハイライト分布から見える関心の重心
3. **テーマ別整理**: 5〜8テーマ。各テーマにハイライト要約（ページ番号付き）と、対応する**ユーザーメモを太字の「あなたのメモ」として引用**
4. **あなたのメモから見えること**: メモ群を横断して読み、ユーザーの視点・関心・変化を2〜4点言語化する（ここがこのスキルの核。要約ではなく解釈を書く）
5. **実践アクション候補**: チェックボックス形式で5〜7個。ユーザーの文脈（マネジメント・コーチング学習など）に接続する
6. **残った問い**: 本やメモが開いたままにした問いを2〜4個

### フェーズ3: 仕上げ
1. ファイルを保存し、コミット・プッシュする（コミットメッセージ例: `Add reading reflection: <書名>`）
2. チャットには全文を貼らず、要点（テーマ一覧＋メモからの気づき＋アクション）を短く提示し、ファイルパスを案内する

## 注意
- CLAUDE.mdの秘匿ルールに従う。メモに顧客名・実名等が含まれる場合は匿名化してからファイルに書く
- ユーザーのメモは原文尊重で引用する（勝手に丸めない）
- 書籍の内容の推測で穴埋めしない。読み取れた範囲だけで構成する
