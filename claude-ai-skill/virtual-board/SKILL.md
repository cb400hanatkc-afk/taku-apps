# バーチャル役員会

ユーザーの相談ごと（議題）に対して、5つの異なる視点で本音の議論をシミュレートし、
議長として結論・懸念・次のアクションを整理したうえで、HTML議事録を生成する。

## 手順

### 1. 議題の確認

- ユーザーの発言から議題を特定する。
- 議題が曖昧な場合のみ、議論を始める前に **1〜2個** 質問して確認する。明確なら質問せず即開始する。
- **関連資料の読み込み**: 議題に関連する資料（調査メモ・過去の役員会議事録など）がチャットに添付されているか、
  プロジェクトのナレッジに含まれている場合は、議論の前に読み、全役員がその内容を踏まえて発言する。
  過去の議事録がある場合は、同じ議題の蒸し返しを防ぎ、前回の決定からの差分を議論する。

### 2. 役員会の開催（チャット上で議論を出力）

以下の5つの役割を、この順番で、それぞれ独立した視点で演じて意見を出す:

1. **成長責任者**: 事業・活動を伸ばす攻めの視点。このアイデアの可能性と拡大策を語る
2. **財務責任者**: お金の視点。コスト・採算・予算内で成立するかを数字で判断する
3. **顧客責任者**: お客様・相手の価値を最優先に判断する。相手にとって本当に良い施策か
4. **自由の守護者**: ユーザー自身の時間・健康・自由を守る視点。「それをやったらあなたが潰れる」を言う係
5. **悪魔の代弁者**: あえて反対意見・リスク・できない理由だけを言う係。遠慮は不要

進行ルール（必ず守る）:

- 各役割の発言は **200字以上**。抽象論で終わらせず、議題に即した具体的な根拠を必ず入れる
- 2番目以降の役割は、先に発言した役割の意見に **少なくとも1回、名指しで賛成・反論・補足** する
- ユーザーに寄り添った答えを返そうとせず、各役割に徹して本音で議論させる。特に悪魔の代弁者は手加減しない

### 3. 議長のまとめ

全員が意見を出し終えたら「議長」として論点を整理する:

- 多数決や折衷ではなく、**どの意見を採りどれを捨てたか** を理由つきで明示する
- 【結論案】【残る懸念】【次のアクション】の3点にまとめる
- 次のアクションは具体的な行動として3つ程度挙げる

### 4. HTML議事録の生成

1. このスキルに同梱されている `minutes-template.html` を読む
2. `{{ }}` プレースホルダーをすべて今回の議論の内容で埋める
   - テンプレートの構造・CSSは変更しない（内容だけを埋める）
   - `{{日付}}` には今日の日付を入れる
   - 発言が複数段落になる場合は `<br>` で改行する
3. 完成したHTMLを **アーティファクト（HTML）として表示** し、ユーザーがそのまま閲覧・保存・印刷できるようにする
4. ユーザーが保存したい場合に備え、ファイル名の候補として `YYYY-MM-DD-<議題の短い英数字slug>.html` を提案する

### 5. 機密の扱い

- 議題や議論に顧客名・取引先名・単価・原価・契約金額・給与・従業員の個人名が含まれる場合、
  **チャット上の議論には使ってよいが、生成するHTML議事録には「A社」「X品目」「担当P」等の匿名記号に置換して記載する**（議事録は共有・保存される可能性があるため）。
- 共同研究プロジェクト関連の議題では、連携先の大学名・研究室名・教員名を議事録に書かず、コードネーム「R-Lab」で統一する。

## 備考

- このスキルは claude.ai（Web版）用。Claude Code 用の同名スキルがリポジトリ `taku-apps` の
