# 共同研究サポートチーム 運用ガイド

共同研究プロジェクト（コードネーム: **R-Lab**）を進めるための、Claude Code 上のサポートチームです。
利用者はリポジトリ所有者本人のみを想定しています。

## 鉄のルール

1. **実名を書かない** — 連携先の大学名・研究室名・教員名は、このディレクトリ配下のファイル・コミットメッセージ・チャット成果物に書かない。すべて「R-Lab」で統一する。
2. **機密情報を入れない** — 顧客名・取引先名・単価・原価・契約金額・個人名は書かない。必要な場合は「A社」「X品目」「担当P」等の匿名記号を使う。
3. **実データはローカル処理** — 顧客名入りの受注実績などの実データは、このリポジトリ・チャットに置かない。チームが作るスクリプト・手順書を使って手元のPCで匿名化してから扱う。

## チーム構成と呼び出し方

チャットで相談すれば秘書（メインのClaude）が自動で振り分けます。直接指名も可能です。

| 担当 | エージェント名 | 相談内容の例 |
|---|---|---|
| 産学連携コーディネーター | `collab-coordinator` | 面談準備、アジェンダ、メール文案、訪問の段取り |
| 生産管理エンジニア | `collab-engineer` | 研究用語の解説、現場課題の構造化、提案の実現性評価 |
| データ準備担当 | `collab-data` | 提供データのフォーマット設計、匿名化ルール、変換スクリプト |
| 契約・リスク担当 | `collab-legal` | NDA・共同研究契約の論点整理、データ提供範囲の線引き |
| 記録・報告担当 | `collab-scribe` | 議事録整形、宿題リスト、経営層向け進捗レポート |

### 使い方の例

> 「共同研究の件。来週R-Labと2回目の打合せ。先方が受注実績データを見たいと言っている。準備を手伝って」

→ 秘書が collab-data（フォーマット・匿名化）と collab-legal（提供範囲の線引き）に振り分け、
統合した「持参物チェックリスト」を出力します。

> 「昨日の打合せメモを貼るから議事録にして」

→ collab-scribe が整形し、実名・機密が混ざっていれば自動で置換して `collab/minutes/` に保存します。

## ディレクトリ構成

```
collab/
├── README.md            ← このファイル
├── actions.md           ← 宿題・アクション一覧（scribe が更新）
├── minutes/             ← 議事録（YYYY-MM-DD-議題.md）
├── reports/             ← 経営層向け報告書
└── templates/           ← 各種テンプレート
    ├── agenda.md        ← 面談アジェンダ
    ├── minutes.md       ← 議事録
    └── data-checklist.md ← データ提供前チェックリスト
```

## 秘匿の限界（正直な注意書き）

- 調査レポートは伏せ字版（`collab/r-lab-research-report.md`）のみをリポジトリに置く。**実名・出典URL入りの完全版は窓口担当が社外非公開で手元保管**し、リポジトリ・チャットには再アップロードしない。
- 作業ブランチ名に実名由来の文字列が含まれる場合がある。マージ後はブランチを削除すること。
