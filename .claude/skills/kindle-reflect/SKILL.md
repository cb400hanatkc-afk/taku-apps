# Kindle読書リフレクション

Kindleでハイライトした箇所・メモのスクリーンショットを受け取り、
「この本をどう捉えたか」を整理し、内省とその後のアクションにつなげる。

ゴールは要約ではなく **「自分がどう変わるか」** を言語化すること。

## 手順

### 1. スクリーンショットの受け取りと文字起こし

- ユーザーからスクリーンショット（複数可）を受け取る。Read ツールで画像を読み、以下を抽出する:
  - **ハイライト箇所**: ハイライトされた本文（長い場合は趣旨を保った短縮引用にする。引用は1箇所あたり2〜3文まで、それ以上は要約に置き換える）
  - **ユーザーのメモ**: Kindleのメモ欄に書かれた本人のコメント（これが内省の最重要素材。一言一句できるだけ忠実に拾う）
  - **位置情報**: ページ番号・位置Noが読み取れれば記録する
- 書名・著者がスクショから読み取れない場合のみユーザーに確認する。読み取れたら質問せず進める。
- 画像が不鮮明で読み取れない箇所は、推測で埋めず「[判読不能]」と明示してユーザーに補完を求める。

### 2. 本の概要と捉え方マップの作成（チャット上に出力）

まず **本の概要** を3〜5文で書く。内容は:

- どんな本か（題材・著者の立場）
- 著者の中心主張は何か
- 誰に向けた本か

概要は一般知識ではなくハイライトから再構成し、知識で補う場合は推測と分かる書き方をする。
ノート保存時もこの概要を最初のセクションに置く。

続けて、文字起こしした素材を次の観点で整理して提示する:

1. **テーマ分類**: ハイライトを3〜5個のテーマに分ける。テーマ名は本の章立てではなく「ユーザーが反応したポイント」でつける
2. **著者の主張 × 自分の反応**: テーマごとに「著者が言っていること」と「ユーザーのメモから読み取れる反応（共感・違和感・疑問）」を対にして並べる
3. **反応の偏りの指摘**: どこに多くハイライトし、どこにメモを書いたかから、「この本のどこに強く反応しているか」を1〜2文で言語化して返す

### 3. 内省の問いかけ（対話）

マップを提示したあと、**質問は最大3つ** に絞って投げかける。テンプレ的な質問を並べず、
ユーザーのメモの内容から個別に組み立てる。使える型:

- 「◯◯のメモに『△△』と書いていますが、これは今の仕事・生活のどの場面を思い浮かべて書きましたか？」（具体化）
- 「著者は◯◯と言っていますが、あなたのメモは逆方向に見えます。どちらが本音ですか？」（矛盾の指摘）
- 「この本を読む前と後で、変わった前提はありますか？」（ビフォーアフター）

CLAUDE.md の「過度な同調の禁止」をここでも適用する。ユーザーの解釈が本文と食い違っていると思えば、根拠を添えて指摘する。

### 4. アクションへの変換

内省の回答を受けて、アクションを **1〜3個** に絞って提案し、ユーザーと合意する。各アクションは:

- **行動**: 「意識する」「心がける」は禁止。翌週に実行したか検証できる粒度にする（例:「次回の朝会で◯◯を試す」）
- **期限またはトリガー**: いつ/どんな場面でやるか
- **本との紐づけ**: どのハイライト・内省から出たアクションか1行で書く

3個に絞れないほど出た場合は、ユーザーに優先順位を決めてもらう。

### 5. 保存（Notionへ出力）

1. このスキルと同じディレクトリにある `note-template.md` を読み、内容を埋める
2. Notionの「著書の要約」ページ（page_id: `1b77d06b-8cc5-8000-b439-de41dd15d67a`）の配下に、
   **書名をタイトルにしたサブページ** として作成する（`mcp__Notion__notion-create-pages` を使用。
   parent は `{"type": "page_id", "page_id": "1b77d06b8cc58000b439de41dd15d67a"}`）
   - 作成前に同名の本のサブページがないか「著書の要約」ページを fetch して確認する。既にあれば新規作成せず、既存ページに追記する（`notion-update-page`）
   - 既存サブページのタイトル慣習（例: 「著書　◯◯」または書名そのまま）に合わせる
3. 合意したアクションを `reading/actions.md` の表に追記する（振り返りはリポジトリ側で横断管理する）
4. 作成したNotionページのURLをユーザーに伝える
5. Notion MCPが使えない環境の場合のみ、フォールバックとして `reading/notes/YYYY-MM-DD-<書名slug>.md` に保存し、次回Notionに転記する旨を伝える

### 6. 振り返り（2回目以降）

- 同じ本のスクショを追加で受け取ったら、新規ページを作らずNotionの既存ページに追記する
- 「読書アクションの振り返り」と言われたら `reading/actions.md` を開き、未完了アクションの状況を1件ずつ確認して状態を更新する

## 機密・著作権の扱い

- 本文の引用は必要最小限（各2〜3文まで）にとどめ、ノートは要約と自分の言葉を主体にする
- ユーザーのメモに顧客名・取引先名・金額・従業員の個人名が含まれる場合、チャット上では使ってよいが、
  保存するノートには「A社」「担当P」等の匿名記号に置換して記載する（CLAUDE.md 秘匿ルール準拠）
- R-Lab関連の内容がメモに出てきた場合も同様に、実名をノートに書かずコードネームで統一する
