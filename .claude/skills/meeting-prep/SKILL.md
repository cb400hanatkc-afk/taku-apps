# R-Lab 面談準備パック

R-Labとの面談・打合せの準備一式を1回で揃える。theme-research / paper-research の調査結果、
バーチャル役員会の戦略決定、collabチームの資料を1つのアジェンダに束ねる結節点となるスキル。

## 手順

### 1. 状況確認（最大2問）

- 面談の回数（第N回）と「今回決めたいこと」をユーザーの発言から特定する。不明な場合のみ質問する。

### 2. 材料の収集（リポジトリ内を確認）

以下を読み、面談に関連する内容を拾う。存在しないものは飛ばす:

- `collab/actions.md` — 未完了の宿題（前回宿題の確認セクションに反映）
- `collab/minutes/` — 直近の議事録（前回の流れ・未決事項）
- `board/minutes/` — 役員会での戦略・方針の決定事項（当社スタンスの根拠）
- `research/` および `research/papers/` — 動向調査・文献調査（話題の裏付け、先方の研究への言及材料)
- `.claude/skills/production-mgmt-implementation/SKILL.md` — 現状課題・プロジェクト現在地・共同研究テーマ案

### 3. 成果物の生成

`collab/templates/agenda.md` の形式をベースに、以下をまとめて1ファイルにする:

1. **アジェンダ** — 前回宿題の確認 → 本題（今回決めたいこと）→ 次回日程・宿題、の骨格に時間配分を付ける
2. **当社スタンスメモ** — 役員会決定・調査結果に基づく「譲れる点・譲れない点・持ち帰る点」
3. **想定Q&A（5件程度）** — collab-engineer の視点で、先方から聞かれそうな質問と回答案。
   研究用語には現場の言葉での言い換えを添える
4. **持参物チェックリスト** — データ提供が議題に含まれる場合は `collab/templates/data-checklist.md`
   への参照と、collab-legal 視点での提供範囲の注意を1段落添える

### 4. 保存

- `collab/agenda-YYYY-MM-DD.md` として保存し、チャットにも全文を表示する
- 面談後に議事録ができたら actions.md を更新するよう、末尾にリマインドを1行入れる

## 鉄のルール（CLAUDE.md準拠）

- 実名（大学名・研究室名・教員名）を成果物に書かない。「R-Lab」「先生」等で統一する
- 顧客名・単価・原価・契約金額は「A社」「X品目」等の匿名記号に置換する
- 出席者は役割表記（窓口担当、生産管理担当 等）で記載する
