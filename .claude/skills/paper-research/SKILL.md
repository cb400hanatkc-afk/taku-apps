# 文献・論文リサーチ

指定されたテーマの学術文献（論文・紀要・学会発表・書籍）を調査し、**先行研究の地図**を作る。

## theme-research との使い分け

| | theme-research | paper-research（このスキル） |
|---|---|---|
| 対象 | ニュース・プレスリリース・業界動向 | 査読論文・紀要・学会発表・レビュー論文 |
| 鮮度 | 直近6ヶ月重視 | 古典的な基礎文献も含める（引用数・影響度重視） |
| 用途 | 動向のインプット | R-Lab共同研究の準備、研究テーマの裏付け |

## 調査手順

1. WebSearch で日英両方の検索語を使い、以下を組み合わせて検索する:
   - 日本語文献: 「site:jstage.jst.go.jp」「site:cir.nii.ac.jp」「〜 論文」「〜 学会」
   - 英語文献: 「site:arxiv.org」「site:semanticscholar.org」「"survey" / "review"」+ 専門用語
   - 分野の定番を掴むため「〜 review」「〜 サーベイ」でレビュー論文を最優先で探す
2. 有望な文献は WebFetch で要旨（アブストラクト）まで確認し、タイトルだけで判断しない。
3. 各文献について記録する: 著者 / 発表年 / 掲載誌・会議名 / 要旨2〜3行 / 引用数（分かれば）/ URL
4. 全体を俯瞰して「この分野の研究は大きく〜派と〜派に分かれる」のような**地図**を1段落で書く。

## 出力フォーマット

### 1. 分野の地図（1段落）
### 2. 主要文献リスト（5〜10件、重要度順）
- 各件: 書誌情報 + 要旨2〜3行 + 「なぜ重要か」1行
### 3. 自社・共同研究との接点
- `.claude/skills/production-mgmt-implementation/SKILL.md` の「現状課題」「共同研究テーマ」と突き合わせ、
  どの文献がどの課題・テーマの裏付け・参考になるかを対応づける
### 4. 未解決領域（研究の空白）
- 先行研究が薄い領域を挙げる。共同研究テーマの独自性の根拠になる

## 成果物の保存

- `research/papers/YYYY-MM-DD-<テーマの短いslug>.md` に保存する（ディレクトリがなければ作成）
- 同じテーマの過去調査があれば「前回からの追加文献」を冒頭に明記する
- 保存後、チャットにも全文を表示する

## 秘匿ルール（CLAUDE.md準拠・保存ファイルに適用）

- **R-Labの研究者本人・研究室の論文を調査した場合**: チャット上では実名・論文原題を扱ってよいが、
  **保存するファイルには著者名・論文原題・所属を書かない**（検索による特定を防ぐため）。
  「R-Lab主宰者の2024年の論文（生産スケジューリング分野）」のように内容ベースの表記に置き換え、要旨の要約のみ記載する。
- R-Labと無関係な一般文献は、通常どおり書誌情報を記載してよい。
- 出典URLは実際にアクセスして確認したもののみ記載する（捏造しない）。確認できない文献は「未確認」と明記する。
